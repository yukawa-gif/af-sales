## 確度ランクの定義

| ランク | 意味 |
|--------|------|
| 売上 | 確定・入金済み |
| 決定 | 受注確定・未入金 |
| A | 高確度見込み |
| B | 中確度見込み |
| C | 低確度見込み |
| 失注 | 失注確定 |

**ダッシュボードの「売上合計」はExcelの`=SUMIF(決定)+SUMIF(売上)`相当**（決定+売上の合算）。

---

## 粗利の計算式

```
粗利（単月）= 売上 - 費用
総粗利 = 粗利 × 月数 × コース数 × 数量
```

インポート時は**粗利を直接入力**（売上・費用は参考値）。
GASが利益を計算するのではなく、入力値の粗利をそのまま使う設計。

## 重要な技術的知見

### GAS日付型のバグ対策
スプレッドシートのDate型は `instanceof Date` で判定できない（GAS環境のバグ）。
必ずduck typingで判定する：
```javascript
typeof v.getFullYear === 'function'
```

### localStorageは使わない
以前localStorageをキャッシュに使っていたが廃止。GASスプレッドシートを唯一のデータソースとする。

### file://プロトコルは使えない
`file://`でdashboard.htmlを開くとGASへのfetchがセキュリティエラーになる。
必ずローカルサーバー経由で動作確認：
```bash
python -m http.server 8080
# → http://localhost:8080/dashboard.html
```

### GASシートの列順序は変えない
`getMaster()`と書き込み系関数で列インデックスを直接参照しているため、
列を追加する場合は末尾に追加すること。
